## 1. О проекте

«REZISTORKA» — это платформа для музыкантов, где можно создавать треки онлайн. Готовую музыку можно отправить в
«свалку», где каждый трек живёт за счёт лайков сообщества, а лучшие попадают в топ недели.

Проект даёт свободу творчества без цензуры и ограничений. Здесь нет лейблов и редакторов —
только музыка и те, кто её оценивает. Подходит для независимых музыкантов, подкастеров
и всех, кто хочет поделиться своим звуком.

## 2. Функциональные возможности

- Регистрация и авторизация с подтверждением email
- Создание треков во встроенном аудиоредакторе AudioMass
- Загрузка готовых аудиофайлов (MP3, WAV, OGG)
- «Свалка» — случайный трек, лайки, рейтинг
- Топ треков и топ исполнителей на главной
- Фильтр треков по жанрам
- Поиск по трекам, исполнителям, жанрам и поджанрам
- Страницы жанров и поджанров со списком треков и плейлистом
- Плейлисты: создание, добавление треков, изменение порядка, удаление
- Встроенный аудиоплеер с плейлистом и переключением треков
- Смена светлой и тёмной темы
- REST API с аутентификацией по API-ключу
- Адаптивный дизайн для мобильных устройств

### 4.2. Логика работы основных модулей

**Авторизация:**

- Flask-Login управляет сессиями
- `login_user()` / `logout_user()` — вход/выход
- `@login_required` защищает приватные страницы

**API:**

- Flask-RESTful обрабатывает эндпоинты
- `check_api_key()` проверяет заголовок `X-API-Key`
- Для обычных пользователей API-ключ подставляется в шаблоны

**Плеер:**

- Клиентский класс `RadioPlayer` (HTML5 Audio)
- Поддерживает плейлисты и одиночное воспроизведение
- Синхронизирует иконки воспроизведения

**База данных:**

- SQLAlchemy ORM
- Все сессии закрываются через `with` (контекстный менеджер)
- `joinedload()` для жадной загрузки связанных объектов

### 4.3. Модели базы данных

**User**

- `id`, `username`, `email`, `password` (хеш), `confirmed`, `modified_date`

**Track**

- `id`, `title`, `genre_id`, `subgenres`, `file_path`, `users_id`
- `likes_count`, `views_count`, `intop_count`, `created_at`

**Genre**

- `id`, `title`

**Dump**

- `id`, `track_id`, `dumped_at`

**Playlist**

- `id`, `user_id`, `title`, `created_at`

**PlaylistTrack** (связующая таблица)

- `id`, `playlist_id`, `track_id`, `order`

**ApiKey**

- `id`, `key`, `name`, `created_at`, `is_active`

> Ключевое поле `intop_count` обновляется раз в неделю планировщиком APScheduler.

```bash
git clone https://github.com/your-username/RadioSvobodi.git
cd RadioSvobodi
```

```bash
pip install -r requirements.txt
```

Создайте файл .env в корне проекта:

```env
SECRET_KEY=ваш_секретный_ключ
ADMIN_API_KEY=ваш_административный_api_ключ
MAIL_PASSWORD=пароль_приложения_yandex
```

> В первый раз можно сгенерировать ADMIN_API_KEY автоматически — раскомментируйте соответствующий код в main.py и
> запустите один раз.

### 5.4. Инициализация базы данных

База данных создаётся автоматически при первом запуске.

python test.py

python main.py

### 6.3 Пример запроса (curl)

#### Получить все треки

```bash
curl -X GET http://127.0.0.1:5000/api/tracks \
-H "X-API-Key: ваш_ключ"
```

#### Создать трек

```bash
curl -X POST http://127.0.0.1:5000/api/tracks \
-H "X-API-Key: ваш_ключ" \
-H "Content-Type: application/json" \
-d '{"title":"Моя песня","genre_id":1,"file_path":"/music.mp3","users_id":1}'
```

#### Поставить лайк

```bash
curl -X POST http://127.0.0.1:5000/api/tracks/1/like \
-H "X-API-Key: ваш_ключ" \
-H "Content-Type: application/json" \
-d '{"user_id": 1}'
```

#### Пример запроса (Python)

```python
import requests

headers = {"X-API-Key": "ваш_ключ"}

# Получить треки
response = requests.get("http://127.0.0.1:5000/api/tracks", headers=headers)

# Создать трек
data = {
    "title": "Моя песня",
    "genre_id": 1,
    "file_path": "/music.mp3",
    "users_id": 1
}
response = requests.post("http://127.0.0.1:5000/api/tracks",
                         json=data,
                         headers=headers)
```

### 7.3. Связи между таблицами

```
users ──┬── tracks (один-ко-многим: пользователь создаёт треки)
├── playlists (один-ко-многим: пользователь создаёт плейлисты)
├── likes (один-ко-многим: пользователь ставит лайки)
└── api_keys (один-ко-многим: у пользователя может быть несколько ключей)

tracks ──┬── dump (один-к-одному: трек в свалке)
├── playlist_tracks (один-ко-многим: трек в плейлистах)
└── likes (один-ко-многим: лайки трека)

genres ─── tracks (один-ко-многим: жанр может быть у многих треков)
playlists ─── playlist_tracks (один-ко-многим: плейлист содержит много треков)

```

## 8. Интерфейс

### 8.1. Общая стилистика

- **Тёмная тема** (по умолчанию) с возможностью переключения на **светлую**.
- Акцентный цвет: `#484AD5` (фиолетовый).
- Шрифты: **Inter** (основной), **Courier Prime** (для заголовков и акцентов), **Bytesized** (для цифр).
- Фоны: `#0D0D0D` (основной), `#1A1A1A` (карточки, шапка, шторка).

### 8.2. Структура страниц

#### Главная (`/`)

- Приветственный блок с логотипом и кнопками «Создать» / «На свалку».
- Разделитель «ТОПЫ» (полоса с текстом).
- Кнопка фильтра по жанрам (циклическое переключение).
- **Топ треков** (5 позиций): номер, кнопка воспроизведения (треугольник), название трека, исполнитель, сердечко с
  лайками.
- **Топ исполнителей** (5 позиций): номер, аватарка, имя исполнителя, сердечко с суммой лайков.
- Вертикальные декоративные картинки (меняются при смене темы).

#### Свалка (`/dump`)

- Центральная картинка-мусорка с текстом.
- Плашка трека (появляется после клика): треугольник (воспроизведение), сердечко с лайками.
- Рандомный трек из всех, добавленных в свалку.

#### Поиск (`/search`)

- Одна строка поиска (ищет по трекам, исполнителям, жанрам, поджанрам).
- Навигация по разделам (якоря) при наличии результатов.
- Результаты разделены на блоки: «ТРЕКИ», «ИСПОЛНИТЕЛИ», «ЖАНРЫ» (ссылки), «ПОДЖАНРЫ» (ссылки).

#### Страницы жанра и поджанра (`/genre/<id>`, `/subgenre/<name>`)

- Заголовок (например, «Жанр: Рок»).
- Список всех треков этого жанра/поджанра (номер, треугольник, название, исполнитель, лайки).
- Кнопка возврата к поиску.
- Все треки собираются в плейлист плеера (последовательное воспроизведение).

#### Плейлисты (`/playlists`)

- Список созданных плейлистов (название, дата создания).
- Кнопка «+ Создать».
- У каждого плейлиста — кнопка удаления ✕.

#### Страница плейлиста (`/playlist/<id>`)

- Заголовок плейлиста и кнопка «Удалить плейлист».
- Список треков с элементами управления:
    - Стрелочки (▲▼) для изменения порядка.
    - Кнопка воспроизведения (треугольник).
    - Название, исполнитель, лайки.
    - Кнопка удаления трека из плейлиста ✕.

#### Загрузка трека (`/upload-track`)

- Форма: название, жанр (select2 с поиском), поджанры (текст), аудиофайл (.mp3, .wav, .ogg).
- Кнопка «КИНУТЬ В СВАЛКУ».

#### Аудиоредактор (`/editor`)

- Iframe с [AudioMass](https://audiomass.co) (полноценный аудиоредактор).
- Кнопка «КИНУТЬ В СВАЛКУ» ведёт на страницу загрузки.

### 8.3. Мини-плеер

- Фиксирован внизу страницы.
- Содержит: картинку обложки (заглушка), название трека, кнопки управления (◀, ▶/⏸, ▶▶), кнопку «+» (добавить в
  плейлист).
- При клике на треугольник в любом треке — плеер начинает его играть.
- При клике на «+» — появляется выпадающий список плейлистов пользователя (выбери, куда добавить).

### 8.4. Шторка (боковое меню)

- Открывается по кнопке ☰ в шапке.
- Выезжает сверху на 300px.
- Содержит ссылки: «Профиль», «Свалка», «Настройки».
- Закрывается по крестику или клику вне.

### 8.5. Смена темы

- Кнопка-круг в шапке.
- При клике переключает класс `light-theme` на `body`.
- Выбор темы сохраняется в `localStorage`.
- Декоративные картинки на главной меняются в зависимости от темы.

### 8.6. Адаптив

- Планшеты (≤ 1024px): уменьшены отступы, кнопки.
- Телефоны (≤ 768px):
    - Вертикальное расположение топов.
    - Уменьшены шрифты, отступы, кнопки.
    - Горизонтальные списки становятся вертикальными.
    - Плеер и футер подстраиваются под экран.
- Маленькие телефоны (≤ 480px): ещё меньше шрифты и отступы.

## 9. Планы и дальнейшее развитие

### 9.1. Что уже реализовано

- Полноценная система аккаунтов (регистрация, вход, подтверждение email).

- Адаптивный дизайн.
